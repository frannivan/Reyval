2. Descripción: Registro y Validación de Pagos

ObjetivoRegistrar los abonos del cliente y mantener la transparencia financiera.
ActoresCliente (Iniciador), Contabilidad (Validador), Sistema (Pasivo).
IniciaciónEl Cliente realiza una transferencia o depósito y accede a "Mis Pagos".
Flujo de Mensajes 1. El Cliente sube la imagen del comprobante y el monto.
2. El Sistema marca el pago como "Pendiente de Validación".
3. El Sistema notifica a Contabilidad para su revisión en banco.
4. Contabilidad marca el pago como "Validado".
5. El Sistema actualiza el saldo del cliente y genera el recibo PDF.
Flujos Alternativos A1: Pago Rechazado: Contabilidad anota el motivo (ej. Saldo insuficiente) y el Cliente debe re-subir.
Valor FinalSaldo actualizado en el ERP y comprobante PDF oficial para el cliente.
