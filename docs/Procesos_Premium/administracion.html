2. Descripción: Reset de Contraseña (Transversal)

ObjetivoPermitir a cualquier usuario recuperar el acceso a su cuenta de forma segura y autónoma.
ActoresUsuario (Iniciador), Sistema (Pasivo).
IniciaciónEl Usuario hace click en "¿Olvidó su contraseña?" en la pantalla de login.
Flujo de Mensajes 1. El Usuario proporciona su correo electrónico institucional/personal.
2. El Sistema valida la existencia del usuario y genera un token temporal.
3. El Sistema envía un enlace de recuperación vía email.
4. El Usuario accede al enlace y define su nueva contraseña.
5. El Sistema invalida el token y confirma el cambio.
Flujos Alternativos A1: Email no registrado: El Sistema no confirma si existe, por seguridad, pero no envía nada.
A2: Token expirado: El Sistema solicita iniciar el proceso de nuevo (validez 30 min).
Valor FinalAcceso restaurado con credenciales actualizadas y registro en el log de seguridad.
