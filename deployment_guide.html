3. TUTORIAL: Cómo crear y editar archivos (Nano) 📝

En el servidor no tienes ratón. Usaremos el editor Nano. Aquí tienes los "Superpoderes" que necesitas memorizar:

Para ABRIR o CREAR un archivo:

sudo nano /ruta/del/archivo

UNA VEZ DENTRO:

  • Usa las Flechas para moverte.
  • Copia el texto desde tu Mac (Cmd+C) y pégalo en la terminal (Cmd+V).

PARA GUARDAR Y SALIR (¡Vital!):

  1. Presiona Ctrl + O (letra O) → Esto significa "WriteOut" (Guardar).
  2. Presiona Enter → Para confirmar el nombre del archivo.
  3. Presiona Ctrl + X → Para salir del editor.

4. Configuración del Servicio (Systemd)

Cada aplicación necesita un archivo de "servicio" para funcionar en segundo plano.

Ejemplo para AbTech (Repetir para cada app)

1. Crea el archivo de servicio:

sudo nano /etc/systemd/system/abtech.service

2. Pega este contenido (ajusta los puertos según la lista de abajo):

[Unit]
Description=Backend AbTech
After=syslog.target

[Service]
User=ubuntu
# IMPORTANTE: Cambia el puerto aquí si es necesario (-Dserver.port)
ExecStart=/usr/bin/java -Dserver.port=8082 -jar /opt/abtech/app.jar
WorkingDirectory=/opt/abtech
SuccessExitStatus=143
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

3. Guarda y Sal:

Usa Ctrl+O, Enter, Ctrl+X.

  • BarberShop: Puerto 8080 (Usa -Dserver.port=8080)
  • Amulets: Puerto 8081 (Usa -Dserver.port=8081)
  • AbTech: Puerto 8082 (Usa -Dserver.port=8082)
  • Reyval: Puerto 8083 (Usa -Dserver.port=8083)
  • LaCanchita: Puerto 8084 (Usa -Dserver.port=8084)
  • LoteMaster: Puerto 8085 (Usa -Dserver.port=8085)

4. Recarga Systemd (para que reconozca los nuevos archivos):

sudo systemctl daemon-reload

5. Configuración Nginx (Conectar Internet -> App)

Como ya tienes configuraciones existentes, simplemente vamos a agregar las nuevas Apps a tu archivo actual.

1. Abre tu configuración:

sudo nano /etc/nginx/sites-available/default

2. BORRA todo el contenido (Ctrl+K repetidametne) y PEGA esta configuración completa y unificada:

server {
    listen 80;
    server_name _;

    # 1. BARBERSHOP (Raíz)
    location / {
        root /var/www/barbershop;
        try_files $uri $uri/ /index.html;
    }
    location /api/ {
        proxy_pass http://localhost:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 2. AMULETS OF SIAM (/amulets)
    location = /amulets { return 301 /amulets/; }
    location /amulets/ {
        alias /var/www/amulets/;
        try_files $uri $uri/ /amulets/index.html;
    }
    location /api/amulets/ {
        proxy_pass http://localhost:8081/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 3. ABTECH (/abtech)
    location = /abtech { return 301 /abtech/; }
    location /abtech/ {
        alias /var/www/abtech/;
        try_files $uri $uri/ /abtech/index.html;
    }
    location /api/abtech/ {
        # El backend tiene context-path=/abtech, así que recibe /abtech/api/...
        # Aquí mapeamos /api/abtech/ -> /abtech/api/
        rewrite ^/api/abtech/(.*)$ /abtech/api/$1 break;
        proxy_pass http://localhost:8082;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 4. LACANCHITA (/lacanchita)
    location = /lacanchita { return 301 /lacanchita/; }
    location /lacanchita/ {
        alias /var/www/lacanchita/;
        try_files $uri $uri/ /lacanchita/index.html;
    }
    location /lacanchita/api/ {
        # El frontend llama /lacanchita/api/...
        # El backend escucha en /lacanchita/api/...
        proxy_pass http://localhost:8083/lacanchita/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 5. CASAVIDA (/reyval)
    location = /reyval { return 301 /reyval/; }
    location /reyval/ {
        alias /var/www/reyval/;
        try_files $uri $uri/ /reyval/index.html;
    }
    location /api/reyval/ {
        # Backend Reyval (8084) por defecto escucha en /api/reyval/ ?? 
        # Ojo: No pusimos context-path en Reyval.properties, solo cambiamos puerto.
        # Si Reyval Controller es @RequestMapping("/api/reyval"), entonces ok.
        proxy_pass http://localhost:8084/api/reyval/; 
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 6. LOTEMASTER (/lotemaster)
    location = /lotemaster { return 301 /lotemaster/; }
    location /lotemaster/ {
        alias /var/www/lotemaster/;
        try_files $uri $uri/ /lotemaster/index.html;
    }
    location /api/lotemaster/ {
        proxy_pass http://localhost:8085/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

3. Verifica y Reinicia Nginx:

sudo nginx -t   # (Debe decir "syntax is ok")
sudo systemctl restart nginx
⚠️ NOTA SOBRE ANGULAR: Para que las apps funcionen en subdirectorios (ej: misitio.com/reyval), el archivo index.html de Angular debe tener <base href="/reyval/">. Si ves la pantalla en blanco, es posible que debamos ajustar ese detalle en el build.

6. Cómo Alternar Apps (Gestión de RAM) 🧠

Recuerda la regla de oro: Máximo 2 apps encendidas.

Comandos para Controlar Apps (Backend)

Concepto Clave:
- Frontend (Web): No se "apaga". Son archivos estáticos en Nginx (como imágenes) y no consumen RAM. Siempre están visibles.
- Backend (Java): Es el motor que procesa datos. Este SÍ consume RAM (~300MB c/u).

Para "apagar" una app y ahorrar memoria, solo necesitas detener el servicio del Backend.
App Encender (Start) Apagar (Stop) Ver Estado
BarberShop sudo systemctl start barbershop sudo systemctl stop barbershop sudo systemctl status barbershop
Amulets sudo systemctl start amulets sudo systemctl stop amulets sudo systemctl status amulets
AbTech sudo systemctl start abtech sudo systemctl stop abtech sudo systemctl status abtech
Reyval sudo systemctl start reyval sudo systemctl stop reyval sudo systemctl status reyval
LaCanchita sudo systemctl start lacanchita sudo systemctl stop lacanchita sudo systemctl status lacanchita
LoteMaster sudo systemctl start lotemaster sudo systemctl stop lotemaster sudo systemctl status lotemaster

Ejemplo: Cambiar de Apps activas

# 1. Liberar memoria (Apagar todo lo que no uses)
sudo systemctl stop barbershop amulets abtech reyval lacanchita lotemaster

# 2. Encender solo las 2 que necesitas (Ej: Reyval y LoteMaster)
sudo systemctl start reyval lotemaster

# 3. Validar que arrancaron
check-apps

8. ACTUALIZACIONES (Cómo subir cambios nuevos) 🚀

Cuando hagas cambios en el código en tu Mac y quieras verlos en el servidor, sigue estos pasos:

A) Actualizar el FRONTEND (Diseño/UI)

1. En tu Mac (Carpeta frontend):

# Generar la versión de producción
npm run build

# Empaquetar
tar -czvf reyval-ui.tar.gz dist/browser

2. Subir al servidor:

scp -i /Users/franivan/Documents/ProyectosWeb/AbTech/ssh-key-2026-01-09.key /Users/franivan/Documents/ProyectosWeb/Reyval/frontend/reyval-ui.tar.gz ubuntu@143.47.101.209:/tmp

3. En el Servidor (SSH):

# Limpiar carpeta actual
sudo rm -rf /var/www/reyval/*

# Descomprimir nuevos archivos (Ruta absoluta del archivo subido)
sudo tar -xzvf /tmp/reyval-ui.tar.gz -C /var/www/reyval --strip-components 2

# Permisos (opcional pero recomendado)
sudo chown -R www-data:www-data /var/www/reyval

B) Actualizar el BACKEND (Lógica/Java)

1. En tu Mac (Carpeta backend):

# Generar el archivo JAR
mvn clean package -DskipTests

2. Subir al servidor:

scp -i /Users/franivan/Documents/ProyectosWeb/AbTech/ssh-key-2026-01-09.key target/*.jar ubuntu@143.47.101.209:/tmp/reyval.jar

3. En el Servidor (SSH):

# Mover a la carpeta final (Sobrescribir el anterior)
sudo mv /tmp/reyval.jar /var/www/reyval/backend.jar

# Reiniciar el servicio para que use el nuevo código
sudo systemctl restart reyval

9. Troubleshooting / Solución de Problemas 🔧

ps -ef | grep java

Ver si los puertos 808x están escuchando:

sudo netstat -tulpn | grep java

Ver contenedores Docker rebeldes (si usaste Docker alguna vez):

sudo docker ps
sudo docker stop [ID]
